## Quick Setup

Get up and running with Bugster CLI in just a few commands.

### 1. Initialize Your Project & Authenticate

```bash
cd your-project-directory
bugster init
```

This command will:
- Prompt for your API key. You can find it in your [Bugster dashboard](https://dashboard.bugster.dev).
- Create a `.bugster` directory for configuration and tests.
- Ask for your application's base URL (e.g., `http://localhost:3000`).

### 2. Generate Your First Tests

```bash
bugster generate
```

Bugster analyzes your codebase and creates test specifications in the `.bugster/tests` directory.

<Tip>
  Use `bugster generate --show-logs` to see what Bugster discovers in your code.
</Tip>

### 3. Run Your Tests

```bash
bugster run
```

For your first run, try:
```bash
bugster run --stream-results
```

    <Note>
  Make sure your application is running on the base URL you specified (e.g., `http://localhost:3000`).
    </Note>

## Common Scenarios

**Testing a Local Development Server:**
```bash
# Start your dev server first
npm run dev

# In another terminal
bugster run --base-url http://localhost:3000
```

**Testing a Staging Environment:**
```bash
bugster run --base-url https://staging.yourapp.com
```

**Debugging Test Failures:**
```bash
# Run with detailed output
bugster run --show-logs --stream-results

# Run without headless mode to see what's happening
bugster run
```

**Updating Tests After Code Changes:**
```bash
# Update tests for code changes
bugster update

# Run updated tests
bugster run
```

## Typical Workflow

```bash
# 1. One-time setup: initialize and authenticate
bugster init

# 2. Generate initial tests
bugster generate

# 3. Run tests
bugster run

# 4. After making code changes
bugster update
bugster run

# 5. Share with team (optional)
bugster sync
```
